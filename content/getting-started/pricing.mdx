## How Pricing Works

Palactix charges **per account per publish**, not per API call.

### Example

If you make one API call that publishes to multiple accounts:

```json
POST /v1/publish
Authorization: Bearer your_token

{
  "accounts": [
    "acc_instagram_123",
    "acc_linkedin_456",
    "acc_tiktok_789",
    "acc_facebook_012",
    "acc_twitter_345"
  ],
  "content": {
    "text": "Product launch today!"
  }
}
```

**You are charged:** 5 accounts × $0.01 = **$0.05**

---

## Why Per Account?

When you publish to multiple accounts, Palactix makes separate API calls to each platform:

- Validates OAuth tokens
- Checks platform rate limits
- Handles errors independently
- Logs each publish event

Publishing to multiple accounts requires **multiple platform API calls**, so pricing reflects actual infrastructure usage.

## Read Operations

**All read operations are free**

Examples include:

- GET /v1/accounts
- GET /v1/accounts/[id]/analytics
- GET /v1/users
- Other GET endpoints

Only publish operations are charged.

## Free Tier Details

### How does the free tier work?

Every developer account gets **100 free publish operations every month**.

### What happens after the 100 free publishes each month?

You can add a payment method and continue using pay-as-you-go pricing.

### Can I test without a credit card?

Yes. The free tier requires **no payment method**.

---

## FAQ

### What counts as a publish?

A publish is counted **per account**.

Publishing to three accounts counts as **three publish operations**.

### Are failed publishes charged?

No. Only successful publishes are counted.

### Is there a monthly minimum?

No. If you publish nothing, you pay nothing.

### Are read operations really free?

Yes. All GET requests are free.

# Introduction

**Palactix** is a unified social publishing infrastructure designed for agencies and developers.

It provides a single API layer for managing social accounts and publishing content — along with an optional white-label scheduler built on top of that same API.

Palactix is API-first.  
The scheduler is simply one implementation of it.

---

## Two Ways to Use Palactix

Palactix can be used in two different ways, depending on your needs.

### As a Unified API (For Developers)

Developers can integrate directly with the Palactix API to:

- Build custom scheduling tools
- Create internal dashboards
- Power automation workflows
- Enable AI-driven publishing
- Integrate social publishing into existing systems

The API provides a consistent interface across supported platforms, allowing you to manage accounts and publish content without building separate integrations for each provider.

---

### As a White-Label Scheduler (For Agencies)

Agencies can use the built-in scheduler to:

- Manage multiple clients
- Handle team workflows
- Publish across platforms
- Operate under custom domains
- Maintain workspace isolation

The scheduler uses the same unified API available to developers.  
There is no separate internal system.

---

## The BYO (Bring Your Own) OAuth Model

Palactix does not provide shared platform application credentials.

Instead, each developer or agency connects their own platform applications (for example, Meta, LinkedIn, or other supported providers).

This means:

- You retain control of your platform apps.
- You control your API keys.
- Your connections are not tied to a shared Palactix application.
- You reduce vendor-level lock-in.

Palactix manages the orchestration and infrastructure —  
you control the platform credentials.

A detailed explanation of this model is available in the [Authentication & Ownership](/docs/auth) section.

---

## Architecture Overview

At a high level, Palactix consists of:

- A unified API layer
- App-based authentication and permission control
- Multi-tenant workspace isolation
- Event logging and observability
- An optional scheduler built on top of the same API

The API is stateless and token-based.
